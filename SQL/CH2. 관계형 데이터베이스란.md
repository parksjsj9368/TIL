## 2.1 관계형 데이터베이스 개념과 특징

<br>

- 관계형 모델 : 테이블이라는 2차원 구조에 데이터를 성격에 맞게 분리해 가급적 중복되지 않게 저장하고, 필요한 데이터는 테이블 간에 관계를 맺어 추출하는 것
- RDBMS : 관계형 모델을 기반으로 데이터베이스를 관리하는 시스템

<br>

### 데이터 관리 입장

1. 데이터 저장 관점 : 필요한 데이터만 저장, 가급적 중복된 데이터는 저장하지 않는다

2. 데이터 조회 관점 : 원하는 데이터를 손쉽게 찾기

3. 데이터 수정 관점 : 특정 데이터만 골라내 원하는 값으로 수정

4. 데이터 삭제 관점 : 원하는 데이터만 골라내 삭제

<br>

<br>

### RDBMS 특징

<br>

1. **데이터 저장소인 테이블**

RDBMS는 2차원 형태의 테이블이라는 구조에 데이터를 저장

로우(=행) : 지정된 컬럼 데이터로 이루어진 하나의 묶음

컬럼(=열) : 테이블 속성

<br>

2. **관계 맺기**

데이터 성격에 맞게 테이블을 여러 개로 분리하고, 분리한 테이블 간에 연결고리 역할을 하는 컬럼을 두어 이 컬럼을 이용해 관계를 맺는 것(= 데이터베이스 모델링)

모델링은 정답이 없으며 데이터베이스 시스템의 성격에 따라 달라지며, 모델링을 어떻게 하느냐에 따라 천차만별

<br>

3. **키 컬럼**

RDBMS에서는 데이터 무결성을 확보하고자 테이블에 반드시 하나의 키가 되는 컬럼을 두도록 권고한다

데이터 무결성 : 데이터의 정확성을 보장해 올바른 데이터를 유지하는 것

키 컬럼(=기본 키) : 한 테이블에서 유일한 값으로 로우를 식별하는 키, 중복 값 불가능

참조 키 : 다른 테이블의 기본 키와 연결되는 키, 중복 값 가능

<br>

4. **트랜잭션 처리**

최종 입력 작업이 완료되기 전에 오류가 발생하면 작업 전 상태로 되돌리고, 입력 작업이 정상적으로 끝난 후에 모든 변경 내용을 최종 적용한다

트랜잭션 처리 : 원자성, 일관성, 고립성, 지속성

<br>
